# ucv_ati2_reto5

Práctica Laboratorio 2 - Mi práctica TDD y pruebas unitarias 

Mis conclusiones:

Durante este reto pude aplicar el ciclo completo de Desarrollo Guiado por Pruebas (TDD) de una manera estructurada. Primero construí un controlador simple y generé las pruebas iniciales que validaban respuestas básicas. Luego evolucioné el proyecto para manejar una lista real de categorías, creando el modelo Category, ajustando el controlador y mejorando las pruebas para validar tamaño, id, nombre y descripción de cada elemento.

Posteriormente incorporé la capa de servicio (CategoryService), lo que permitió separar correctamente las responsabilidades del proyecto bajo el patrón MVC. Con esta separación fue posible introducir Mockito, simulando el comportamiento del servicio y permitiendo pruebas unitarias más limpias y aisladas.

Finalmente, verificar los tests en verde y revisar los reportes HTML generados me permitió confirmar que el código funcionaba como se esperaba en cada fase del proceso.

En general, este reto me ayudó a comprender mejor cómo aplicar TDD en un proyecto real, cómo mantener una estructura de código ordenada y modular, y cómo las pruebas unitarias bien diseñadas mejoran la calidad del software desde el principio.
